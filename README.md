## Project Overview
This is the final project for the SQL Developer Internship. The goal was to design and implement a relational database for an e-commerce platform to manage customers, products, orders, and payments.

## Key Features
- **Normalized Schema:** Designed tables for Customers, Products, Orders, Order_Items, and Payments.
- **Advanced SQL:** Used JOINS, GROUP BY, and Aggregate functions for sales reporting.
- **Automation:** Implemented a **Database Trigger** to automatically adjust stock levels when a sale is made.
- **Stored Procedures:** Created reusable procedures for retrieving customer history.

## Files in this Repository
- `schema.sql`: DDL scripts to create the database structure.
- `data.sql`: DML scripts to populate the database with sample data.
- `Reports.sql`: SQL queries for business analysis, including Views and Triggers.
- `Project Title_ Online Retail Sales Database System.pdf`: Detailed documentation of the project.
- `EER Diagram.jpg`: Visual representation of the database schema.

## How to Run
1. Run `schema.sql` to create the tables.
2. Run `data.sql` to insert the sample data.
3. Run queries from `Reports.sql` to view the analysis.
